# 即时制重构 — Phase 执行计划

> **文档性质**：Phase TODO（可执行跟踪文档）  
> **基线日期**：2026-02-26  
> **代码基线**：前端 JS 10,583 行 · 后端 Python 29,735 行 · CSS 4,272 行 · HTML 4,929 行（共 ~49.5k 行）  
> **决策结论**：采用 **PixiJS v8 渐进式混合制**——格子级半即时移动 + 回合制事件触发 + HD-2D 后处理

## 0. 决策摘要与约束

### 0.1 核心决策

| 决策项 | 结论 | 理由 |
|--------|------|------|
| 目标层级 | L1.5 格子级半即时（移动平滑化 + 事件回合制） | LLM 延迟 2-30s 与即时制 16ms 帧预算存在根本冲突，纯 ARPG 不可行 |
| 前端引擎 | **PixiJS v8**（WebGL 2D） | 专注 2D、API 简洁、Filter/粒子原生支持、CDN 引入无需构建工具链、与 DOM UI 可共存 |
| HD-2D 还原度 | 70-80%（PixiJS + GLSL 后处理模拟景深/光影） | 95%+ 需 Three.js，开发成本翻 2-3 倍，不值当 |
| 改造策略 | 渐进式 P0→P1→P2→P3 串行主线，P4/P5 并行 | 避免一次性重写 80% 前端代码的风险 |
| 不变更项 | 后端 FastAPI 架构、LLM 异步集成、JSON 存档、DOM 侧栏/对话框/调试面板 | 这些系统与渲染引擎无耦合 |

### 0.2 硬约束

- **LLM 延迟不可压缩**：叙述 2-8s、事件选择 3-10s、地图生成 5-30s → 只能做异步预加载+推送，不能阻塞帧循环
- **无持续帧动画美术资源**：当前仅有静态 sprite sheet（`CharacterClass.png` 102×136px/职业）和 `monster_default.png`（所有怪物共用）
- **DOM UI 必须保留**：侧栏（属性/背包/任务）、`EventChoiceManager.js` 对话框、`DebugManager.js` 调试面板 → PixiJS Canvas + DOM Overlay 共存
- **存档向前兼容**：`save_version=3` / `equipment_schema_version=2` → 新字段必须有默认值

## P1：视觉升级与交互迁移

> **目标**：地图渲染完全迁移到 PixiJS，添加 HD-2D 光影后处理，角色精灵替换。  
> **预计工时**：80-160h（单人 2-4 周）  
> **前置依赖**：P0 完成  
> **交付物**：DOM 地图代码可安全移除 / Bloom+GodRay 后处理 / 角色基础帧动画

### P1.1 分层渲染架构

- [ ] 创建 PixiJS 容器层级：

```
Stage
├── FloorLayer      ← 地板纹理 + overlay 叠加（替代 floor_layers.js + floor_layers.css）
├── ShadowLayer     ← 墙壁/实体阴影投射
├── EntityLayer     ← 玩家精灵 + 怪物精灵 + NPC
├── EffectLayer     ← 粒子特效 + 技能动画（替代 EnhancedEffectsManager.js DOM 部分）
├── FogLayer        ← 战争迷雾（替代 fog_canvas.js 独立 Canvas）
└── UILayer         ← 血条 + 浮动伤害数字
```

- [ ] `FloorLayer`：基础地板纹理 + overlay 混合（`PIXI.Sprite` 叠加替代 CSS `background-image` 多层）
- [ ] `ShadowLayer`：墙壁旁投射简单阴影精灵（半透明黑色矩形，偏移 2-3px）

### P1.2 角色精灵系统

- [ ] `CharacterSprites.js` CSS sprite → PixiJS：从 `CharacterClass.png`（510×408px）裁剪各职业帧 → `PIXI.Texture.from()` + `Rectangle`
- [ ] 角色站立动画：2 帧呼吸感（原帧 + 轻微 Y 偏移帧），`PIXI.AnimatedSprite` 0.5fps 循环
- [ ] 角色 4 方向行走动画：**需要新美术资源**（最低 4 方向 × 4 帧 = 16 帧/职业）
  - 临时方案：单帧 + Tween 位移 + 左右翻转模拟方向
- [ ] 怪物精灵：从 `monster_default.png` → `PIXI.Sprite`，根据 `behavior` 字段染色（`sprite.tint`）
- [ ] 地形图标：`door_icon.png` / `stairs_up.png` / `stairs_down.png` / `trap_icon.png` / `treasure_chest.png` → `PIXI.Sprite`

### P1.3 地图交互迁移

### P1.4 缩放与拖拽迁移

- [ ] 缩放：[`MapZoomManager.js`](static/MapZoomManager.js:1) CSS `transform: scale()` → `stage.scale.set(zoom)` + `stage.pivot` 锚点

### P1.5 迷雾系统迁移

## P2：半即时移动系统

> **目标**：角色移动从"瞬移到目标格"→"平滑滑行到目标格"，支持键盘连续输入。  
> **预计工时**：60-120h（单人 2-3 周）  
> **前置依赖**：P1 完成  
> **交付物**：`static/GameLoop.js` + `static/InputManager.js` + 平滑移动 + 行走动画

### P2.1 游戏主循环

- [ ] 创建 `static/GameLoop.js`：`requestAnimationFrame` 帧循环 + `deltaTime` 管理
- [ ] 分离 `update(dt)` 逻辑帧与 `render()` 渲染帧（逻辑帧可固定 60Hz，渲染帧跟随显示器）
- [ ] 与 PixiJS `app.ticker` 整合（使用 `app.ticker.add()` 而非独立 rAF）
- [ ] 游戏状态机：`IDLE` / `MOVING` / `ANIMATING` / `WAITING_EVENT` / `PAUSED`

### P2.2 平滑移动

- [ ] 角色移动：从当前格中心 → 目标格中心的 Tween 线性插值（`PIXI.Ticker` + 自定义 lerp）
- [ ] 移动速度可配置：`config.game.player_move_speed`（默认 200ms/格，即 5 格/秒）
- [ ] 移动锁定：移动动画播放中 `gameState.playerMoving = true`，禁止新输入
- [ ] 方向更新：移动方向决定精灵朝向（左/右翻转 `sprite.scale.x = ±1`）
- [ ] 怪物移动同样添加平滑动画（与玩家移动**串行**播放：玩家动画结束 → 怪物逐个移动）

### P2.3 键盘/手柄输入

- [ ] 创建 `static/InputManager.js`：统一键盘/鼠标/手柄输入管理
- [ ] WASD / 方向键 → 格子级别移动（按下→计算目标格→执行移动）
- [ ] 按住连续移动：`keydown` 持续中的重复间隔（首次 0ms，后续 200ms）
- [ ] 输入队列：移动动画中缓存最后一个方向输入，动画结束后自动执行
- [ ] 可选：Gamepad API 支持（`navigator.getGamepads()`）
- [ ] 输入模式切换：键盘模式下隐藏 [`DirectionButtonManager.js`](static/DirectionButtonManager.js:1) 移动端方向按钮

### P2.4 碰撞与事件触发

- [ ] 保持格子级碰撞检测（不引入像素碰撞）：目标格 `terrain_type === 'wall'` → 拒绝移动
- [ ] "到达目标格"回调：触发事件检查（陷阱/宝箱/楼梯/故事事件）→ 复用现有 [`LocalGameEngine.js`](static/LocalGameEngine.js:1) 事件处理逻辑
- [ ] 事件触发后切换状态机到 `WAITING_EVENT`，暂停移动输入
- [ ] 怪物碰撞检测保持不变（格子级邻近判定）

### P2.5 怪物动画化 AI

### C. HD-2D 技术分析

歧路旅人核心技术：UE4 2D 精灵 + 3D 场景 + 后处理管线。Web 端 PixiJS 近似实现路径：

- **景深**：多层 `Container` 视差滚动 + 近景层 `PIXI.BlurFilter`
- **体积光**：`@pixi/filter-godray`（光柱角度/密度可调）
- **泛光**：`@pixi/filter-bloom`（strength 2-4）
- **色调**：`PIXI.ColorMatrixFilter` 暖色/冷色预设
- **像素完美缩放**：`PIXI.SCALE_MODES.NEAREST`（不模糊）
